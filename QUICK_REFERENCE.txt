FONT LOADING STRATEGY
================================================================================

Current Pattern (Line 28):
  <link rel="preload" href="...css...?display=swap" as="style"
        onload="this.onload=null;this.rel='stylesheet'">

How it works:
  1. Preload CSS (non-blocking)
  2. onload callback converts to stylesheet
  3. display=swap shows system font immediately
  4. Custom fonts apply when ready

Benefits:
  ✓ No FOIT (Flash of Invisible Text)
  ✓ No FOUT (Flash of Unstyled Text)
  ✓ Minimal CLS from font swap (~0.02-0.04)
  ✓ Fonts don't block initial paint
  ✓ Fallback to sans-serif if CDN down

Result: EXCELLENT - No optimization needed

COMMON ISSUES (PREVENTIVE)
================================================================================

CLS Causes to Avoid:
  ✗ Images without width/height → USE: Always set dimensions
  ✗ Late-loading ads → USE: Reserve space with fixed aspect ratio
  ✗ Font loading FOUT → USE: display=swap in font URL
  ✗ Dynamic content → USE: Pre-allocate space or use fixed-position
  ✓ Current site: No CLS issues detected

LCP Blockers to Avoid:
  ✗ Render-blocking CSS → USE: Inline critical CSS, defer rest
  ✗ Render-blocking JS → USE: Async/defer attributes, load fonts non-blocking
  ✗ Large images → USE: Preload, optimize, lazy load
  ✓ Current site: All optimized

INP Causes to Avoid:
  ✗ Long JavaScript tasks → USE: Break into <50ms chunks
  ✗ Heavy event handlers → USE: Debounce, requestAnimationFrame
  ✗ Blocking DOM queries → USE: Cache references
  ✓ Current site: Minimal, efficient JS
